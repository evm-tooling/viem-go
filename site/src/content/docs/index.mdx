## Overview

viem-go brings the developer experience of [viem](https://viem.sh) to the Go ecosystem. If you're familiar with viem in TypeScript, you'll feel right at home.

<Tabs>
  <TabItem label="viem-go">
```go
// 1. Import modules.
import (
    "github.com/ChefBingbong/viem-go/client"
    "github.com/ChefBingbong/viem-go/client/transport"
    "github.com/ChefBingbong/viem-go/chain/definitions"
)

// 2. Set up your client with desired chain & transport.
c, _ := client.CreatePublicClient(client.PublicClientConfig{
    Chain:     definitions.Mainnet,
    Transport: transport.HTTP("https://eth.llamarpc.com"),
})
defer c.Close()

// 3. Consume an action!
blockNumber, _ := c.GetBlockNumber(context.Background())
```
  </TabItem>
  <TabItem label="viem (TypeScript)">
```typescript
// 1. Import modules.
import { createPublicClient, http } from 'viem'
import { mainnet } from 'viem/chains'

// 2. Set up your client with desired chain & transport.
const client = createPublicClient({
  chain: mainnet,
  transport: http(),
})

// 3. Consume an action!
const blockNumber = await client.getBlockNumber()
```
  </TabItem>
</Tabs>

<div class="section-header">
<h2>Why Go?</h2>
<p>viem-go is designed for teams and projects that need Ethereum tooling in the Go ecosystem and have familiarity with the viem typescript library.</p>
</div>
<div class="card-grid four-col">
<div class="info-card">
<h3>Backend Services</h3>
<p>Build high-performance blockchain indexers, APIs, and microservices</p>
</div>
<div class="info-card">
<h3>CLI Tools</h3>
<p>Create command-line utilities for wallet management and contract interactions</p>
</div>
<div class="info-card">
<h3>Infrastructure</h3>
<p>Power validators, relayers, and other blockchain infrastructure</p>
</div>
<div class="info-card">
<h3>Cross-platform</h3>
<p>Compile to a single binary for any OS without dependencies</p>
</div>
</div>
</div>

<div class="full-width-section">
<div class="section-header">
<h2>Features</h2>
<p>viem-go supports all main features from the original viem typescript library. Every feature was built using the same syntax, method-names and patterns so that the developer firendly nature still remains.</p>
</div>
<div class="card-grid four-col">
<div class="info-card">
<h3>JSON-RPC Abstractions</h3>
<p>High-level APIs over the JSON-RPC API to make your life easier</p>
</div>
<div class="info-card">
<h3>Smart Contracts</h3>
<p>First-class APIs for interacting with Smart Contracts</p>
</div>
<div class="info-card">
<h3>Ethereum Terminology</h3>
<p>Language closely aligned to official Ethereum terminology</p>
</div>
<div class="info-card">
<h3>Multiple Accounts</h3>
<p>Private Key, Mnemonic, and HD Wallet account types</p>
</div>
<div class="info-card">
<h3>Safe Numerics</h3>
<p>Go's native big.Int for safe numeric operations</p>
</div>
<div class="info-card">
<h3>ABI Utilities</h3>
<p>Encoding, decoding, and inspection utilities for ABIs</p>
</div>
<div class="info-card">
<h3>Token Bindings</h3>
<p>Pre-built bindings for ERC20, ERC721, and ERC1155</p>
</div>
<div class="info-card">
<h3>Battle Tested</h3>
<p>Test suite running against Anvil for real-world compatibility</p>
</div>
</div>
